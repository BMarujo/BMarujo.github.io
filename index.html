Introduction

Building a comprehensive 5G data collection and visualization solution for Smart Cities.

Android Development

The primary goal of the Android application is to provide a flexible tool for collecting a wide range of metrics directly from the User Equipment (UE) on a private 5G network. This app is designed to gather critical data such as signal quality (RSSI), bandwidth, and connection stability metrics like iPerf and Speedtest results. Unlike existing commercial solutions, our app integrates seamlessly with the Aveiro Tech City Living Lab (ATCLL) platform, allowing collected metrics to flow into a unified Smart City visualization environment for deeper insights.

Web Development

The web application focuses on the real-time and historical visualization of data collected across the ATCLL's private 5G network. Leveraging React and Next.js for dynamic and spatial data representation, this platform displays network metrics from gNodeBs, UEs, and other ATCLL infrastructure on a city-wide map. The web app will serve as a comprehensive dashboard, displaying aggregated quality metrics to assess connectivity across different city regions and aiding in ongoing Smart City optimizations.

Timeline

Project Progress by Week

  • First Week

    Tools Presentation

    Explored NewATCLL. Presentation of tools such as: React, Next.js, ShadcnUI, and GeoJSON data format.

    Started mockups in Figma to highlight data, such as number of UEs and antenna locations.

    Set up MQTT server for testing Android data transmission and began Android Studio project setup.

  • Second Week

    Metrics and Mockups

    Fist experience with metrics visualization tools, working with MQTT for data export.

    Explored RabbitMQ and Mosquitto for MQTT implementation, assessing the options.

    Started iPerf/Speedtest independent app for metrics collection.

    Exploration of a first version data format reference for the upcoming API integration.

  • Third Week

    MQTT and Data Handling

    Established MQTT connection between Network Survey app and PC (as MQTT Broker).

    Began storing MQTT data in CSV for future MongoDB integration.

    Discussed project architecture models, including website structure and content.

  • Fourth Week

    Mockup Clarifications and Testing

    Addressed questions on mockup designs and clarified the scope within ATCLL development, databases, and data visualization tools.

    MQTT data conversion tests from CSV to GeoJSON.

    Set up GitLab access and discussed workshop opportunities for project collaboration.
